## 第一章 项目概览

### 1.1 项目背景

河北省自2021年起推行"3+1+2"新高考改革，本科批次实行**平行志愿96个**（无专业调剂，退档即滑档）。考生面临海量数据（每年约2万条录取记录）、多年横向比对、体检政策合规等挑战。本系统旨在为考生提供智能化、数据驱动的志愿填报辅助决策工具。

### 1.2 核心功能

| 模块 | 功能说明 |
|------|---------|
| **数据管理** | 导入2023~2025年河北省招生Excel数据，自动清洗、去重、入库 |
| **智能推荐** | 基于考生位次，按冲/稳/保三梯度生成96个志愿方案 |
| **合规过滤** | 体检限制（色觉/视力/身高体重）、选科要求自动过滤 |
| **结果导出** | 支持Excel（带颜色格式）、PDF（三段式报告）、DOCX三种格式 |
| **数据查询** | 历史录取数据预览、院校信息查询、一分一段表查询 |

### 1.3 技术栈

| 层次 | 技术选型 |
|------|---------|
| 语言 | Python 3.13 |
| GUI框架 | PyQt6 6.4+ |
| 数据库 | SQLite 3（内嵌，无需安装） |
| 数据处理 | pandas 2.0+、numpy 1.24+ |
| Excel导出 | xlsxwriter 3.1+ |
| PDF导出 | reportlab 4.0+ |
| Word导出 | python-docx |
| Excel读取 | openpyxl 3.1+ |
| 打包工具 | PyInstaller 6.0+ |

### 1.4 数据规模

| 年份 | 科目 | 条数 | 院校数 |
|------|------|------|--------|
| 2023 | 物理 | 16,864 | 1,127 |
| 2024 | 历史 | 6,507 | 1,100 |

### 3.2 admission_data 表

```sql
CREATE TABLE admission_data (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    year            INTEGER NOT NULL,          -- 录取年份
    subject         TEXT NOT NULL,             -- 物理/历史
    batch           TEXT DEFAULT '本科批',
    college_code    TEXT,                      -- 院校代码（注：每年从行号生成，跨年不可用）
    college_name    TEXT NOT NULL,             -- 院校名（已去除[公办]等后缀）
    major_code      TEXT,
    major_name      TEXT NOT NULL,             -- 专业名
    plan_count      INTEGER,                   -- 计划招生人数
    rank_lowest     INTEGER,                   -- 最低录取位次（核心字段）
    score_lowest    INTEGER,                   -- 最低录取分数
    score_avg       REAL,                      -- 平均分
    province        TEXT,                      -- 院校所在省份
    elective_req    TEXT,                      -- 选科要求
    body_restrict   TEXT,                      -- 体检限制说明
    UNIQUE(year, subject, batch, college_name, major_name)
);
```

> ⚠️ **重要**：`college_code` 字段的值是Excel文件中的行序号，各年度不一致，**不能跨年作为院校唯一标识**。跨年匹配时应使用 `college_name`（标准化后）。

### 5.2 多年数据聚合策略

#### 聚合Key设计

```
key = 标准化(college_name) + "||" + 标准化(major_name) + "||" + batch
```

**标准化规则**：
- 去除院校名括号内的城市后缀（如"中国人民大学（北京）"→"中国人民大学"）
- 去除专业名括号内的方向后缀（如"计算机科学与技术（大数据方向）"→"计算机科学与技术"）

> ⚠️ **不能用 college_code**：每年代码实为Excel行号，跨年不一致。

#### 聚合权重

```python
# 各年权重（越新越重要）
YEAR_WEIGHTS = {2025: 0.5, 2024: 0.35, 2023: 0.15}

# 加权平均最低位次
avg_rank = sum(rank * YEAR_WEIGHTS[year] for year, rank in ranks) / sum(YEAR_WEIGHTS[y] for y in years)
rank_std  = std(ranks)  # 位次标准差（稳定性指标）
```

- **修复前**：按key聚合后只有约20%有多年数据
- **修复后**：72.5%有2~3年数据，11,962个专业具有3年完整数据

### 5.3 过滤规则

#### 5.3.1 体检限制过滤

| 条件 | 过滤规则 |
|------|---------|
| 色盲 | 过滤所有色弱+色盲限制专业（美术、绘画、摄影、化学、医学、药学等） |
| 色弱 | 过滤色弱限制专业（化学、化工、药学、生物、医学、公安技术等） |
| 裸眼视力 < 5.0 | 过滤飞行技术、航海技术、消防工程、刑事科学技术、侦查学 |
| 裸眼视力 < 4.8 | 过滤轮机工程、运动训练、民族传统体育 |
| 公安类（男） | 身高 < 170cm 或体重 < 50kg |
| 公安类（女） | 身高 < 160cm 或体重 < 45kg |

过滤方式：优先读取数据库 `body_restrict` 字段，兜底用关键词匹配。

#### 5.3.2 选科要求过滤

```python
# 仅当 required_electives 是 student_electives 的子集时，该专业通过
if not required.issubset(student_electives):
    continue  # 过滤
```

#### 5.3.3 位次范围过滤

```python
rank_upper = student_rank * (1 - rank_reach_pct / 100)  # 冲刺上限（比考生好）
rank_lower = student_rank * (1 + rank_safe_pct / 100)   # 保底下限（比考生差）
```

### 5.4 录取概率计算

```python
def calc_probability(rank_diff, rank_std, years_count):
    # rank_diff = student_rank - avg_rank_lowest
    # 正值 → 偏冲刺（考生位次比专业历史位次差），概率较低
    # 负值 → 偏保底（考生位次比专业历史位次好），概率较高
    
    base_prob = sigmoid(-rank_diff / (rank_std + 2000))
    
    # 数据完整度修正（年数越多越可靠）
    completeness_factor = {1: 0.85, 2: 0.95, 3: 1.0}[years_count]
    
    return base_prob * completeness_factor
```

### 5.5 梯度切分算法（_build_tiers）

```
候选列表（按位次差排序，rank_diff ≈ 0 的专业排在中间）

定义锚点：rank_diff 最接近 0 的专业设为第 30 位

分配规则：
  冲（Tier 1）：第  1 ~  30 位（共30个）
  稳（Tier 2）：第 31 ~  75 位（共45个）
  保（Tier 3）：第 76 ~  96 位（共21个）
  合计：96个志愿
```

### 5.6 综合评分（排序依据）

```python
score = (
    probability       * 0.35 +  # 录取概率
    preference_score  * 0.30 +  # 偏好匹配度（省份+专业）
    stability_score   * 0.20 +  # 位次稳定性（1/rank_std归一化）
    completeness      * 0.15    # 数据完整度（年数/3）
)
```

### 5.7 省份/专业偏好宽松机制

当按偏好过滤后候选不足96个时，系统自动放宽：
1. **第一轮**：仅匹配偏好省份+偏好专业
2. **第二轮**：放宽省份限制（全国院校），仍匹配偏好专业
3. **第三轮**：放宽专业限制，仍在偏好省份
4. **第四轮**：完全无偏好过滤（全量候选）

放宽后的志愿项 `is_relaxed = True`，在结果界面用橙色高亮显示。

### 12.1 常见问题

**Q：程序启动后数据库为空？**  
A：首次运行需在"数据管理"面板导入Excel数据文件。

**Q：推荐结果数量不足96个？**  
A：系统已自动进行4轮放宽匹配，若仍不足说明分数段可匹配专业确实有限，可调大"冲刺位次放宽%"滑块。

**Q：部分专业名显示为橙色？**  
A：橙色表示该专业是在放宽省份或专业偏好限制后才被纳入，可作为参考但不符合主要偏好。

**Q：导出PDF时中文显示为方块？**  
A：系统自动检测系统中文字体，请确保Windows系统已安装宋体(simsun.ttc)或黑体(simhei.ttf)。

**Q：打包后exe文件在其他电脑无法运行？**  
A：目标电脑需安装Visual C++ Redistributable 2019+，或使用NSIS打包含运行时库的安装包。
