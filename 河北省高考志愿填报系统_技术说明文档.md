## 1. 项目概述

### 1.1 背景

河北省2026年高考采用"3+1+2"模式，本科批平行志愿最多可填96个志愿（每个志愿独立院校+专业组合），无专业调剂，退档即滑档。考生面临以下挑战：

- 96个志愿手动填写工作量极大
- 位次换算复杂（原始分→位次→历史数据对比）
- 各专业的体检限制、选科要求各不相同
- 跨年横向比较录取趋势难以人工完成

### 1.2 系统功能

本系统基于2023~2025年河北省官方投档数据（共73,907条），为考生提供：

- **智能推荐**：按冲30/稳45/保21梯度自动生成96个志愿
- **位次驱动**：所有推荐均基于位次而非原始分，更客观准确
- **合规筛选**：自动过滤不满足选科要求、体检条件的专业
- **偏好匹配**：支持专业方向偏好、省份偏好、办学性质过滤
- **多格式导出**：支持导出Excel志愿表和PDF分析报告

## 4. 数据库设计

数据库文件：`data/db/gaokao.db`（SQLite 3）

### 4.1 院校信息表（colleges）

存储全国院校基本信息，由导入时自动维护。

```sql
CREATE TABLE colleges (
    id        INTEGER PRIMARY KEY AUTOINCREMENT,
    code      TEXT NOT NULL UNIQUE,   -- 院校代码（Excel行号，仅当年有效）
    name      TEXT NOT NULL,          -- 院校名称（去除[公办]等性质后缀）
    province  TEXT,                   -- 所在省份
    city      TEXT,                   -- 所在城市
    level     TEXT,                   -- 层次：本科/专科
    type      TEXT,                   -- 类型：综合/理工/师范等
    nature    TEXT DEFAULT '公办',    -- 办学性质：公办/民办/中外合作
    remark    TEXT
);
```

**重要说明**：`code` 字段是Excel中的行号，每年都会变化，**不能跨年使用**。推荐引擎中院校匹配统一使用院校名称而非代码。

### 4.2 专业信息表（majors）

存储专业基本信息（当前主要通过admission_data中内嵌信息使用）。

```sql
CREATE TABLE majors (

    code          TEXT NOT NULL,
    name          TEXT NOT NULL,
    category      TEXT,              -- 学科门类
    subject_req   TEXT DEFAULT '不限',
    elective_req  TEXT DEFAULT '不限',
    body_restrict TEXT,
    remark        TEXT
);
```

### 4.3 投档数据表（admission_data）★ 核心表

存储2023~2025年历年录取投档数据，是推荐引擎的主要数据源。

```sql
CREATE TABLE admission_data (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    year            INTEGER NOT NULL,   -- 年份（2023/2024/2025）
    college_code    TEXT NOT NULL,      -- 院校代码（仅当年有效）
    college_name    TEXT NOT NULL,      -- 院校名称（已去除[公办]等后缀）
    college_nature  TEXT DEFAULT '公办',-- 办学性质：公办/民办/独立学院/中外合作
    major_name      TEXT NOT NULL,      -- 专业名称（含括号后缀，保留原始格式）
    subject_type    TEXT NOT NULL,      -- 科目类型：物理/历史
    batch           TEXT NOT NULL,      -- 批次
    plan_count      INTEGER,            -- 计划招生人数
    actual_count    INTEGER,            -- 实际录取人数
    min_score       REAL,               -- 最低录取分
    max_score       REAL,               -- 最高录取分（部分年份无此字段）
    avg_score       REAL,               -- 平均录取分（部分年份无此字段）
    min_rank        INTEGER,            -- 最低录取位次
    province        TEXT DEFAULT '河北' -- 生源省份
);
CREATE INDEX idx_admission_year_type ON admission_data(year, subject_type, college_code);
```

**`college_nature` 字段说明**（2026-04-03新增）：

- 由importer.py在导入时从院校名称后缀`[公办]`/`[民办]`/`[独立学院]`/`[中外合作办学]`解析得到

## 5. 数据导入模块

模块路径：`src/data/importer.py`

### 5.1 适配的Excel格式

河北省教育考试院发布的Excel文件有以下特点：
- **表头位置**：第2行（行索引2），前两行是大标题和排序说明
- **列名含换行符**：如`"院校\n代号"`、`"投档\n最低\n分"`，需要清洗
- **院校名称后缀**：`[公办]`/`[民办]`/`[独立学院]`/`[中外合作办学]`
- **多Sheet文件**：2023年物理有5个分片Sheet，自动合并；2025年文件含"一分一段"Sheet
- **说明行跳过**：表头后有排序说明行（如"一、二、三"），需识别并跳过

### 5.2 列名映射（COL_MAP）

支持模糊匹配（三轮策略），适配不同年份不同列名写法：

| 内部字段 | Excel可能的列名 |
|---------|--------------|
| college_code | 院校代号、院校代码、学校代码、代码 |
| college_name | 院校名称、学校名称、院校 |
| major_name | 专业名称、专业(类)名称、专业 |
| min_score | 投档最低分、最低分、录取最低分 |
| min_rank | 2025年位次、2024年位次、最低位次 |
| plan_count | 计划数、招生计划 |

### 5.3 院校名称清洗逻辑（`_clean_college_name`）

```python
# 输入示例：
#   "北京城市学院[民办]"         → clean="北京城市学院",     nature="民办"
#   "广东以色列理工学院[中外合作办学]" → clean="广东以色列理工学院", nature="中外合作"
#   "天津外国语大学滨海外事学院[独立学院]" → clean="天津外国语大学滨海外事学院", nature="独立学院"
#   "合肥工业大学(合肥市)"        → clean="合肥工业大学(合肥市)", nature="公办"（括号内非性质标记，保留）
```

处理逻辑：
1. 用正则 `\[([^\]]+)\]` 查找所有方括号内容
2. 判断括号内容是否包含性质关键词（`公办/民办/独立学院/中外合作/合作办学/职业学院`）
3. 若是性质标记：将其从名称中去除，通过 `NATURE_MAP` 标准化
4. 若不是性质标记（如城市名）：保留在院校名称中
5. 返回 `(clean_name, nature)` 元组

### 5.4 表头识别策略（`_preprocess_df`）

```
扫描前10行 → 找含"院校"/"专业"/"代号"关键词的行 → 该行为表头行
设置列名（规范化：去除所有空白字符）
跳过表头后的说明行（特征：含"语数"/"外语"/"首选科目"等，或全是汉字序号）
第一列是4位数字（院校代码格式）即为真实数据行
```

### 5.5 导入流程

```
1. 读取Excel文件，获取所有Sheet名
2. 遍历Sheet：
   - 含"一分一段"关键词 → 调用 _import_score_rank_sheet()
   - 含跳过关键词（说明/备注/封面）→ 跳过
   - 其他 → 调用 _preprocess_df() 解析为DataFrame
3. 合并所有Sheet的DataFrame（去重：按院校名+专业名）
4. 调用 _write_to_db() 批量写入数据库
```

### 5.6 写库逻辑（`_write_to_db`）

```
1. DELETE 同年份+同科目+同批次的旧数据（幂等设计）
2. 遍历每行数据：
   - 清洗院校名称（提取性质标记）
   - 获取院校省份（通过省份映射工具）
   - 组装record tuple（14个字段）
3. executemany 批量INSERT到 admission_data
4. INSERT OR IGNORE 院校信息到 colleges 表
```

### 5.7 一分一段导入（`_import_score_rank_sheet`）

河北省2025年Excel文件中包含"一分一段"Sheet，格式如下：
```
行0：大标题
行1："考生成绩统计表"
行2：分数档次 | 物理(含优惠分) | — | 历史(含优惠分) | —
行3：— | 人数 | 累计人数 | 人数 | 累计人数
行4+：真实数据
```

导入时根据`subject_type`选择对应列（物理：列1/2，历史：列3/4）。

### 6.1 核心数据结构

#### StudentProfile（考生档案）

```python
@dataclass
class StudentProfile:
    estimated_score: int        # 预估总分（不含加分）
    subject_first: str          # 首选科目：物理/历史
    subject_elective1: str      # 再选科目1
    subject_elective2: str      # 再选科目2
    color_vision: str           # 色觉：正常/色弱/色盲
    naked_eye_vision: str       # 裸眼视力（如"5.0"、"4.8"）
    gender: str                 # 男/女
    height_cm: int              # 身高（cm）
    weight_kg: int              # 体重（kg）
    extra_score: int            # 政策加分
    pref_majors: list           # 偏好专业列表（关键词分类）
    pref_provinces: list        # 偏好省份列表
    accept_private: bool        # 是否接受民办（含独立学院）
    accept_joint: bool          # 是否接受中外合作

    rank_2026: int              # 用户手动输入的2026位次（0=未输入）
```

- 当`rank_2026>0`时，严格使用用户输入的位次

### 6.2 数据聚合策略（`_aggregate`）

这是引擎中最关键的设计决策：**每年的院校代码（college_code）实际是Excel行号，跨年不可用**，因此必须用院校名+专业名聚合。

#### 聚合Key设计

```python
# 标准化院校名（去括号城市后缀）
norm_college = re.sub(r'[（(][^)）]*[)）]', '', college_name).strip()
# 标准化专业名（去括号后缀，仅用于分组匹配）
norm_major = re.sub(r'[（(][^)）]*[)）]', '', major_name).strip()

# 聚合key
key = f"{norm_college}|{norm_major}|{batch}"
```

**效果**：72.5%的专业有2~3年数据，11,962个专业有完整3年数据。

#### 聚合统计量

| 统计量 | 计算方式 | 用途 |
|--------|---------|------|
| avg_min_score | 近3年最低分均值 | UI展示（分数参考） |
| avg_rank_3yr | 近3年平均最低位次 | **核心排序依据** |
| rank_std | 近3年位次标准差 | 波动性评估、概率修正 |
| min_rank_latest | 最近年（2025）最低位次 | 锚点排序 |
| data_years | 有数据的年份数 | 可信度修正 |

#### 位次获取逻辑

```python
# 每年优先用数据库中的 min_rank 字段（直接录取位次）
# 若 min_rank 为空，通过最低分反查当年一分一段表
if raw_rank > 0:
    ranks.append(raw_rank)
else:
    score = r.get("min_score") or 0
    if score > 0:
        est_rank = self._get_rank_cached(score, year)  # 反查
        if est_rank < 999999:
            ranks.append(est_rank)
```

### 6.3 位次范围确定（`_get_rank_range`）

系统使用动态位次范围而非固定分差。

#### 手动模式（用户设置了滑块）

```python
# 冲方向滑块 rank_offset_neg：向更好位次浮动N名
rank_min = max(1, student_rank - rank_offset_neg)
# 保方向滑块 rank_offset_pos：向更差位次浮动N名  
rank_max = student_rank + rank_offset_pos
```

#### 自动模式（两个滑块都为0）

根据考生位次段动态调整范围（确保能凑够96条）：

| 位次段 | 冲方向浮动 | 保方向浮动 |
|--------|-----------|-----------|
| ≤5,000 | 3,000 | 12,000 |
| 5,001~20,000 | 9,600 | 36,000 |
| 20,001~80,000 | 19,500 | 78,000 |
| >80,000 | 30,000 | 120,000 |

若候选不足96条，自动扩大范围（每次扩大50%，最多扩大10次）。

### 6.4 多阶段筛选流程（`generate`）

```
原始数据（全部年份）
    │
    ▼ 阶段0：批次过滤（只保留目标批次，如"本科批"）
    │
    ▼ 阶段1：合规筛选（_passes_body_check）
    │         ├─ 色觉检查（色盲/色弱与专业禁忌）
    │         ├─ 裸眼视力检查（<5.0/<4.8限制）
    │         ├─ 身高检查（公安/航海/飞行类）
    │         └─ 体重检查（公安类专业）
    │
    ▼ 阶段A：位次范围确定
    │         ├─ 用户手动设置 → 严格按范围，不扩大
    │         └─ 自动计算 → 动态浮动，候选不足时自动扩大
    │
    ▼ 阶段B：位次筛选（_passes_rank_filter）
    │         过滤avg_rank_3yr在[rank_min, rank_max]范围内的专业
    │
    ▼ 阶段C：省份过滤（_passes_filter，strict_province=True）
    │         若偏好省份候选数 < 96 → 自动放宽省份限制
    │
    ▼ 阶段D：专业偏好过滤（_passes_filter，strict_major=True）
    │         若偏好专业候选数 < 96 → 自动放宽专业限制
    │
    ▼ 计算位次差、录取概率、综合评分
    │
    ▼ 阶段E：梯度分配（_build_tiers）→ 96个志愿
```

### 6.5 梯度分配算法（`_build_tiers`）

这是生成最终96个志愿的核心算法。

#### 算法步骤

**步骤1：全局去重排序**
```python
# 按2025年最低位次从小到大排序（位次小=录取线高=难进）
unique_items.sort(key=lambda x: (min_rank_2025, -composite_score))
```

**步骤2：定位锚点**
```python
# 找2025年最低位次最接近考生位次的专业，作为"刚好能进"的基准线
anchor_idx = min(range(n), key=lambda i: abs(rank_2025[i] - student_rank))
```

**步骤3：按锚点切分**
```
排序后的专业列表（从难到易）：
[... 更难 ... | 锚点(第30位) | ... 更容易 ...]
    ↑                              ↑
   冲（前29条）              稳(后45) + 保(再后21)

即：
  - 序号1~29：冲  （锚点之前的29条，更难录取）
  - 序号30：锚点（标记为"冲"，最后一个冲志愿）
  - 序号31~75：稳  （锚点之后45条）
  - 序号76~96：保  （之后21条，最容易录取）
```

**步骤4：补齐不足**
```python
# 若候选总数不足96，交替从两侧扩展（先保方向，再冲方向）
while len(result) < 96:
    if baowen_end < n:   # 先从保方向补
        result.append(...)
    elif chong_start > 0: # 再从冲方向补
        result.insert(0, ...)
```

**梯度分配比例**：
| 梯度 | 数量 | 比例 | 含义 |
|------|------|------|------|
| 冲 | 30 | 31.25% | 录取线高于考生位次，录取概率<50% |
| 稳 | 45 | 46.88% | 录取线接近考生位次，录取概率约50~70% |
| 保 | 21 | 21.88% | 录取线低于考生位次，录取概率>70% |

### 6.6 录取概率计算（`_calc_prob_by_rank`）

**核心公式（Sigmoid函数）**：

```python
# rank_diff = 考生位次 - 专业近3年平均最低位次
# rank_diff > 0: 考生位次更差（冒险/冲刺）→ 概率低
# rank_diff < 0: 考生位次更好（稳妥/保底）→ 概率高

z = rank_diff / (rank_std + 2000)
prob = 1 / (1 + exp(-z * 1.2))  # Sigmoid

# 数据完整度修正（多年数据更可信）
completeness = min(data_years / 3, 1.0)  # 1年→0.33，2年→0.67，3年→1.0
prob = prob * completeness + 0.5 * (1 - completeness)

# 边界限制：概率在[0.03, 0.97]之间
prob = max(0.03, min(0.97, prob))
```

**参数说明**：
- `rank_std`：位次波动标准差（加2000防除零，平滑极端情况）
- 乘数`1.2`：调节Sigmoid斜率，使概率变化更敏感
- 单年数据（data_years=1）：概率向0.5回归66.7%

### 6.7 综合评分（`_composite_score`）

用于同梯度内的排序，**不影响梯度分配**。

```python
# 各维度权重
score = (
    prob  * 0.35 +  # 录取概率（35%）
    pref  * 0.30 +  # 偏好匹配度（30%）：专业+省份偏好
    stab  * 0.20 +  # 位次稳定性（20%）：位次波动越小越稳定
    years * 0.15    # 数据完整度（15%）：3年数据最可信
)
```

### 6.8 风险等级判定

```python
def _risk_level(prob, rank_std):
    if prob < 0.40 or rank_std > 5000:  # 概率<40%或历史波动大
        return "高"
    elif prob < 0.70:
        return "中"
    return "低"
```

## 11. 关键数据说明

### 11.1 位次 vs 分数

系统**全程使用位次而非原始分**进行推荐计算，原因：
- 位次消除了不同年份试卷难度差异
- 位次直接反映考生在全省的竞争水平
- 投档录取本质上是位次排序而非分数排序

### 11.2 院校代码跨年问题

河北省投档数据Excel中的院校代码（如`0001`、`0742`）**实际是该Excel文件中的行号**，每年重新编排，同一所院校不同年份代码不同。

**系统解决方案**：
- `admission_data.college_nature` 字段在导入时直接从院校名称解析存储，不依赖代码JOIN
- 推荐引擎聚合使用标准化院校名作为key
- 引擎SQL中 `COALESCE(a.college_nature, c.nature, '公办')` 优先使用直接存储的字段

## 14. 已知问题与注意事项

### 14.1 数据局限性

- **2026年数据未出**：推荐基于2023~2025年历史数据，不能预测2026年实际录取结果
- **min_rank字段**：2023~2024年部分记录无位次数据，通过最低分反查估算，可能有误差
- **中外合作专业**：因数量较少（共179条），候选数可能不足，推荐时可能自动放宽省份/专业限制

### 14.2 技术注意事项

- **打包路径**：数据库打包在`_internal/data/db/gaokao.db`，开发时在`gaokao_volunteer/data/db/gaokao.db`
- **SQLite锁文件**：Windows下偶发锁文件残留（.db-wal/.db-shm），程序启动时自动清理
- **院校代码跨年**：严禁在任何跨年逻辑中使用college_code进行JOIN，必须使用college_name
- **college_nature字段**：2026-04-03才新增，旧数据库需重新导入数据才能生效

### 14.3 推荐结果的使用限制

- 录取概率为统计估算值，仅供参考，不代表实际录取可能性
- 同分排序规则（语数和→单科→外语→首选→再选）可能影响实际投档顺序
